# SOP - Printer Troubleshooting

## First Questions

1. Which exact printer or queue is affected?
2. Does the problem affect one user or multiple users?
3. Is the printer powered on and showing an error?
4. Can the user print a Windows test page?
5. Is the correct printer selected?
6. Is the issue printing, scanning, labels, receipts, or all functions?

## Safe Troubleshooting Order

1. Confirm the correct queue and default printer.
2. Check printer power, display, paper, toner, and visible error state.
3. Confirm network or USB connection.
4. Check the Windows print queue for paused or stuck jobs.
5. Clear only the affected user's safe test jobs when appropriate.
6. Restart the Print Spooler only when the impact is understood.
7. Print a test page.
8. Reinstall or remap the approved queue if required.
9. Escalate hardware faults or repeated disconnects.

## Special Rules

- Do not delete another department's printer queue without confirming scope.
- Do not factory-reset a printer as an initial step.
- Do not change static network settings without documentation and approval.
- For police printers, confirm whether the printer supports citations, reports, labels, or ordinary office documents before clearing jobs.
