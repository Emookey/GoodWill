# Pine Ridge Township Borough Building - Client-Specific SOPs

## SOP 1 - Finance Application Slowness

1. Confirm whether one user or multiple finance users are affected.
2. Check Daniel's device record before analysis.
3. Record CPU, memory, disk active time, free space, and disk-health indicators.
4. Confirm application path and network availability.
5. Test a small report before a large report.
6. Document whether slowness is local, network-related, or application-wide.
7. Recommend SSD/RAM upgrade when evidence supports it.

Do not:
- Delete finance data.
- Move the application database.
- Run repair operations without approval.
- assume every delay is caused by the HDD.

## SOP 2 - Tax or Finance Share Access

1. Confirm exact share path.
2. Confirm prior approved access.
3. Check whether issue began after password change.
4. Check stale saved credentials.
5. Verify network access.
6. Confirm group membership.
7. Request written approval before new permission grants.

## SOP 3 - Permit and Receipt Printers

1. Identify label printer versus receipt printer.
2. Confirm power and connection.
3. Check the exact Windows queue.
4. Check USB power management for repeated disconnects.
5. Print an approved test label or receipt.
6. Escalate recurring hardware faults.

## SOP 4 - Code Enforcement VPN

1. Confirm normal internet works.
2. Record exact VPN error.
3. Verify device time.
4. Restart VPN client and approved adapter.
5. Check whether the issue followed sleep/resume.
6. Reboot if safe and approved.
7. Escalate repeated authentication or site-wide failures.

## SOP 5 - Borough Security Event

Unexpected MFA prompts, forwarding rules, or suspicious logins are High priority.

Follow the shared security alert SOP. Do not disable MFA or delay escalation merely because the user denied the prompt.
