# SOP - Shared Drive Access

## First Checks

1. Confirm the exact path or drive letter.
2. Confirm whether the issue affects one user or multiple users.
3. Confirm the user's department and job role.
4. Determine whether the user previously had access.
5. Check whether the issue began after a password change, device replacement, or role change.
6. Test network connectivity before changing permissions.

## Permission Rule

Access must be based on documented role and approval.

A shared file server does not mean all users should have access to all shares.

## Safe Troubleshooting Order

1. Verify the path.
2. Verify network connection.
3. Check whether other approved users can access the share.
4. Check for stale saved credentials.
5. Refresh mapped-drive authentication where appropriate.
6. Confirm group membership and approval.
7. Escalate permission changes when approval is missing.

## Do Not Do

- Do not add a user to a privileged group merely to test.
- Do not grant Borough users access to Police shares without explicit approval.
- Do not grant Police users access to Finance or Tax shares without explicit approval.
- Do not modify or delete files to test access.
